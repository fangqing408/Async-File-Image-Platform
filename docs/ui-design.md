## 一、设计概览与设计原则

- **平台**：Web（桌面端为主，兼容移动端）
- **核心交互范式**：输入框 + 图片预览区 + 发送按钮的经典留言模式
- **全局异常处理**：网络异常顶部横幅、操作失败 Toast 提示

**交互分歧展开**：

⚠️ 检测到交互分歧点：「粘贴」

| 角色 | 预期行为 | 用户代价/体验影响 |
|:---|:---|:---|
| 产品 | 直接粘贴图片并自动上传 | 操作简单，用户无需额外步骤 |
| 前端 | 需要监听 paste 事件，处理剪贴板数据 | 技术实现复杂度较高 |
| 用户 | 预期 Ctrl+V 能像聊天软件一样直接粘贴 | 符合用户直觉，学习成本低 |

**选择方案 A**：直接监听 `paste` 事件，解析剪贴板中的图片数据，自动创建预览并上传。

---

⚠️ 检测到交互分歧点：「弹窗/对话框」

| 角色 | 预期行为 | 用户代价/体验影响 |
|:---|:---|:---|
| 产品 | 上传失败时弹窗提示 | 信息突出，用户不易忽略 |
| 设计 | 使用 Toast 轻提示 + 内联错误状态 | 不打断操作流程，更优雅 |
| 用户 | 希望快速知道错误但不被打断 | 轻度提示更友好 |

**选择方案 B**：使用内联错误状态（图片上显示红色错误图标）+ Toast 文字提示。

**状态详细说明**：

##### 3.1.1 初始状态 (State-Initial)

| 属性 | 描述 |
|:---|:---|
| **状态标识** | `state-initial` |
| **触发条件** | 页面加载完成、留言提交成功后重置 |
| **输入框** | 显示占位文字，边框灰色，光标闪烁 |
| **图片预览区** | 隐藏 |
| **发送按钮** | 灰色禁用状态，文字「发送」 |
| **字数统计** | 显示「0/500」 |
| **辅助提示** | 输入框下方显示灰色小字：`"支持粘贴图片，格式: JPG、PNG、GIF，单张不超过 10MB"` |

**交互行为**：
- 用户可输入文本
- 输入文本后发送按钮变为可点击状态
- 支持 `Ctrl+V` 粘贴操作

---

##### 3.1.2 粘贴后状态 (State-Post-Paste)

| 属性 | 描述 |
|:---|:---|
| **状态标识** | `state-post-paste` |
| **触发条件** | 用户按下 `Ctrl+V` 且剪贴板包含图片数据 |
| **输入框** | 保持可编辑，光标位于图片占位符之后 |
| **图片预览区** | 从下方滑入显示（动画时长 250ms） |
| **预览图** | 最大宽度 200px、最大高度 150px、圆角 `--radius-md`、阴影 `--shadow-md` |
| **预览布局** | 水平排列，可横向滚动 |
| **图片信息** | 每张图片下方显示文件名和大小 |
| **删除按钮** | 每张图片右上角悬浮显示（×），透明背景，悬浮时变为红色背景 |
| **发送按钮** | 灰色禁用状态，文字「上传中...」 |

**交互行为**：
- 粘贴后延迟 300ms 自动触发上传（支持批量粘贴合并）
- 用户可继续输入文本
- 用户可点击删除按钮移除已粘贴的图片
- 支持连续粘贴多张图片

---

##### 3.1.3 上传中状态 (State-Uploading)

| 属性 | 描述 |
|:---|:---|
| **状态标识** | `state-uploading` |
| **触发条件** | 图片上传请求发出后 |
| **输入框** | 保持可编辑状态 |
| **图片预览区** | 显示加载动画 |
| **加载动画** | 预览图上覆盖半透明黑色遮罩（opacity: 0.5）+ 白色旋转加载图标 |
| **进度提示** | 每张图片下方显示上传进度百分比（如：`35%`） |
| **取消按钮** | 每张图片右上角显示取消按钮（✕） |
| **发送按钮** | 灰色禁用状态，文字「上传中...」 |

**交互行为**：
- 上传进度实时更新（每 100ms 更新一次）
- 用户可取消单张图片上传
- 用户可继续输入文本
- 上传失败自动进入失败状态
- 所有图片上传完成后进入成功状态

---

##### 3.1.4 成功状态 (State-Success)

| 属性 | 描述 |
|:---|:---|
| **状态标识** | `state-success` |
| **触发条件** | 所有图片上传完成且返回成功 |
| **输入框** | 保持可编辑状态 |
| **图片预览区** | 显示完整缩略图 |
| **成功标记** | 每张图片右下角显示绿色勾选图标（✓），半透明圆形背景 |
| **图片链接** | 图片已替换为服务器返回的 URL |
| **发送按钮** | 蓝色高亮状态，文字「发送」，可点击 |
| **字数统计** | 显示当前字数（文本 + 图片数量） |

**交互行为**：
- 用户可继续添加更多图片
- 用户可删除已上传的图片
- 用户可点击发送按钮提交留言
- 用户可使用 `Ctrl+Enter` 快捷键提交
- 发送成功后清空输入框并重置为初始状态

---

##### 3.1.5 失败状态 (State-Failure)

| 属性 | 描述 |
|:---|:---|
| **状态标识** | `state-failure` |
| **触发条件** | 图片上传返回错误或超时 |
| **输入框** | 保持可编辑状态 |
| **图片预览区** | 显示错误状态 |
| **失败标记** | 图片上显示红色错误图标（✕），半透明圆形背景 |
| **错误提示** | 图片下方显示红色错误文字（如：`"上传失败，请重试"`） |
| **重试按钮** | 每张失败图片下方显示「重试」按钮，灰色背景，点击触发重新上传 |
| **发送按钮** | 灰色禁用状态 |

**交互行为**：
- 用户可点击重试按钮重新上传（最多重试 3 次）
- 用户可删除失败的图片
- 重试成功后进入成功状态
- 三次重试失败后显示 `"上传失败，建议检查网络或更换图片"`

**异常场景交互预案**：

| 异常场景 | 交互反馈 | 用户后续操作 |
|:---|:---|:---|
| 网络断开 | 顶部滑入红色横幅「网络连接已断开」，Toast 提示「上传失败，网络异常」 | 点击横幅尝试重连，检查网络后重试 |
| 图片超过 10MB | 粘贴时直接拒绝，Toast 提示「图片大小超过限制（最大 10MB）」 | 更换更小的图片 |
| 不支持的格式 | 粘贴时直接拒绝，Toast 提示「不支持的图片格式，请使用 JPG/PNG/GIF」 | 更换为支持的格式 |
| 上传超时（10秒） | 显示红色错误图标，提示「上传超时，请重试」 | 点击重试按钮 |
| 服务器错误（500） | 显示红色错误图标，提示「服务器繁忙，请稍后重试」 | 稍后再试或联系管理员 |

**状态转换表**：

| 当前状态 | 触发事件 | 目标状态 |
|:---|:---|:---|
| 初始状态 | 输入文本 | 初始状态（发送按钮可用） |
| 初始状态 | Ctrl+V 粘贴图片 | 粘贴后状态 |
| 初始状态 | Ctrl+V 粘贴非图片 | 初始状态（正常插入文本） |
| 粘贴后状态 | 自动上传开始 | 上传中状态 |
| 粘贴后状态 | 删除所有图片 | 初始状态 |
| 上传中状态 | 上传完成 | 成功状态 |
| 上传中状态 | 上传失败 | 失败状态 |
| 上传中状态 | 取消上传 | 粘贴后状态 |
| 成功状态 | 添加新图片 | 粘贴后状态 |
| 成功状态 | 删除所有图片 | 初始状态 |
| 成功状态 | 发送留言 | 初始状态（重置） |
| 失败状态 | 点击重试 | 上传中状态 |
| 失败状态 | 删除失败图片 | 成功状态（有其他图片）/ 初始状态（无图片） |

**快捷键说明**：

| 快捷键 | 功能 | 适用状态 |
|:---|:---|:---|
| `Ctrl+V` | 粘贴图片/文本 | 所有状态 |
| `Ctrl+Enter` | 发送留言 | 成功状态 |
| `Esc` | 清空输入框内容 | 初始/成功状态 |
| `Delete` | 删除选中的图片 | 粘贴后/成功/失败状态 |

## 四、全局组件交互规范

| 组件 | 使用场景 | 交互规范 |
|:---|:---|:---|
| Toast 提示 | 操作成功/失败反馈、错误提示 | 顶部居中显示，2秒后自动消失，支持手动关闭 |
| 模态弹窗 | 重要操作二次确认（如清空所有内容） | 居中显示，包含「取消」和「确认」按钮，半透明遮罩 |
| 加载动画 | 图片上传中、表单提交中 | 旋转圆形图标，白色，大小 24px |

## 五、手势与动效约定

| 手势 | 触发场景 | 动效反馈 |
|:---|:---|:---|
| 粘贴（Ctrl+V） | 输入框内 | 图片预览区滑入动画（250ms ease） |

| 发送按钮点击 | 提交留言 | 按钮缩放反馈（95% → 100%） |
